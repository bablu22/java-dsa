## Complexity Cheatsheet

| Algorithm | Time | Space |
|-----------|------|-------|
| Binary Search | O(log n) | O(1) |
| Merge Sort | O(n log n) | O(n) |
| Quick Sort | O(n log n) avg | O(log n) |
| BFS / DFS | O(V + E) | O(V) |
| Dijkstra | O((V+E) log V) | O(V) |
| DP (most) | O(n²) | O(n) or O(n²) |

---

## Interview Tips

1. Always clarify constraints and edge cases before coding
2. Think out loud — communicate your approach
3. Start with brute force, then optimize
4. Write clean, readable code — name variables clearly
5. Analyze time and space complexity before finishing
6. Test your solution with edge cases (empty input, single element, duplicates)
7. Know when to use which data structure
